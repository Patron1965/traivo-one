Prompt: Fältapp-integration med Kinab Core Concept
Driver Core är en mobilapp (React Native/Expo) för fältchauffirer som ansluter till Kinab Core Concept via REST API. Nedan beskrivs alla endpoints, dataformat och funktioner som backend behöver stödja för att appen ska fungera fullt ut.

Autentisering
POST /api/mobile/login

Stödjer två inloggningsmetoder:
PIN-kod: { pin: "1234" } (4-6 siffror)
Användarnamn/lösenord: { username: "erik", password: "pass" }
Svar: { token, user: { id, name, role, resourceId, vehicleRegNo, executionCodes[] } }
Token skickas som Authorization: Bearer <token> i alla efterföljande anrop

GET /api/mobile/orders/:id — Hämta enskild order

PATCH /api/mobile/orders/:id/status

Body: { status: "in_progress" }
Statusflöde: planned → en_route → in_progress → completed (eller deferred/cancelled)
PATCH /api/mobile/orders/:id/substeps/:stepId

Body: { completed: true }
POST /api/mobile/orders/:id/notes

Body: { text: "Kund ej hemma, lämnat vid grind" }

Appen skickar GPS-position kontinuerligt under körning
Positionen lagras i driver_locations-tabell (UPSERT på driver_id)
Planerarvy (webbaserad)
GET /api/planner/drivers/locations

Returnerar alla chaufförers senaste positioner (senaste 24h)
Svar: [{ driverId, driverName, vehicleRegNo, latitude, longitude, speed, heading, status, currentOrderId, currentOrderNumber, updatedAt }]
GET /api/planner/orders?range=today|week

Returnerar ordrar med koordinater för kartvisning
Filtreras på range: "today" (dagens ordrar) eller "week" (veckans ordrar)
Svar: samma orderformat som ovan men med fokus på kartrelevanta fält
GET /planner/map

Webbsida med Leaflet/OpenStreetMap som visar:
Chaufförers realtidspositioner (cirkulära markörer)
Jobbmarkörer (fyrkantiga, färgkodade efter status)
Filterknappar: Idag / Denna vecka / Dölj jobb
Klustrade markörer vid hög densitet
Popup med orderdetaljer vid klick
Väder
GET /api/mobile/weather

Returnerar väderdata för arbetsområdet (Open-Meteo API)
Daglig sammanfattning
GET /api/mobile/summary

Returnerar: { totalOrders, completedOrders, remainingOrders, totalDuration }
AI-funktioner
POST /api/mobile/ai/chat

Body: { message: "Hur hanterar jag farligt avfall?", context: { orderNumber, customerName } }
Svar: { response: "..." } (GPT-svar på svenska)
POST /api/mobile/ai/transcribe

Body: { audio: "base64-audio-data" }
Svar: { text: "transkriberad text" }
POST /api/mobile/ai/analyze-image

Body: { image: "base64-image", context: "Avvikelserapport" }
Svar: { category, description, severity }
