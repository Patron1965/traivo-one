Prompt att klistra in i fältapp-projektet (Driver Core / Kinab Mobile):

Denna fältapp (React Native/Expo) ska kopplas till Kinab Core-backend. Nedan är hela det API som appen ska använda, med exakta endpoints, autentiseringsflöde, dataformat och funktioner.

BASE URL
Appen konfigureras med BACKEND_URL som pekar på Kinab Core-servern (t.ex. https://kinab-core.replit.app).

AUTENTISERING
Alla endpoints (utom /api/mobile/login och /api/mobile/weather) kräver Bearer token i headern:

Authorization: Bearer <token>
Token fås via login och är giltig i 24 timmar. Vid 401-svar ska appen logga ut användaren.

POST /api/mobile/login
Logga in med PIN eller användarnamn+lösenord:

{ "pin": "1234" }
// eller
{ "username": "erik@example.com", "password": "1234" }
// eller
{ "email": "erik@example.com", "pin": "1234" }
Svar:

{
  "token": "...",
  "success": true,
  "user": { "id": "...", "name": "Erik", "role": "driver", "resourceId": "...", "executionCodes": ["SL", "AV"] },
  "resource": { "id": "...", "tenantId": "...", "name": "...", "phone": "...", "email": "...", "homeLatitude": 57.7, "homeLongitude": 11.9, "executionCodes": [] }
}
POST /api/mobile/logout
Logga ut (invaliderar token).

GET /api/mobile/me
Hämta den inloggade resursens fullständiga data.

ARBETSORDRAR
GET /api/mobile/my-orders?date=2026-02-22
Hämtar dagens arbetsordrar för den inloggade resursen. Sorterade efter scheduledStartTime. Varje order returneras berikad med:

object — namn, adress, GPS-koordinater, vad3ord (what3words)
customer — kundnamn, kundnummer
articles — artiklar kopplade till ordern (id, namn, nummer, enhet)
dependencies — beroenden till andra ordrar
syncCount / notificationCount — sync- och notisräknare
GET /api/mobile/orders
Hämtar alla dagens ordrar (alternativt API, returnerar samma berikade format).

GET /api/mobile/orders/:id
Hämtar en specifik order med full berikning.

PATCH /api/mobile/orders/:id/status
Uppdatera orderstatus:

{ "status": "on_site" }
Giltiga statusar: planned, dispatched, on_site, in_progress, completed, failed, cancelled

Backend uppdaterar automatiskt:

actualStartTime när status sätts till in_progress
actualEndTime och completedAt vid completed
Svarar med den uppdaterade ordern.

POST /api/mobile/orders/:id/notes
Lägg till anteckning:

{ "text": "Porten var låst, ringde kund." }
PATCH /api/mobile/orders/:id/substeps/:stepId
Markera delsteg i strukturella uppgifter:

{ "completed": true }
AVVIKELSER
POST /api/mobile/orders/:id/deviations
Rapportera avvikelse:

{
  "type": "blocked_access",  // blocked_access, damaged_container, wrong_waste, overloaded, other
  "description": "Porten blockerad av parkerad bil",
  "latitude": 57.7089,
  "longitude": 11.9746,
  "photos": ["base64..."]
}
Skapar en deviation_report i backend och broadcastar planner-event.

MATERIAL
POST /api/mobile/orders/:id/materials
Logga använt material:

{ "articleId": "...", "quantity": 2 }
// eller
{ "articleNumber": "ART001", "articleName": "Container 660L", "quantity": 1 }
GET /api/mobile/articles?search=container
Sök artiklar. Returnerar max 50 stycken: { id, articleNumber, name, unit, category }.

SIGNATUR
POST /api/mobile/orders/:id/signature
Spara kundsignatur (skapar protokoll):

{ "signature": "data:image/png;base64,..." }

  "orderId": "...",
  "articleTypes": ["kärl"],
  "checklists": [{
    "templateId": "...",
    "name": "Kärltömning standardkontroll",
    "articleType": "kärl",
    "questions": [
      { "id": "q1", "text": "Är kärlet skadat?", "type": "boolean" },
      { "id": "q2", "text": "Fyllnadsgrad", "type": "select", "options": ["25%","50%","75%","100%"] }
    ]

POST /api/mobile/orders/:id/inspections
Skicka ifylld inspektion:

{
  "inspections": [
    { "category": "kärl", "status": "ok", "issues": [], "comment": "Allt bra" },

  ]

GPS & POSITION

Skicka GPS-position (anropas regelbundet, t.ex. var 15:e sekund):

{
  "latitude": 57.7089,
  "longitude": 11.9746,
  "speed": 45,
  "heading": 180,
  "accuracy": 5,
  "currentOrderId": "...",
  "currentOrderNumber": "WO-2026-001",
  "vehicleRegNo": "ABC123",
  "driverName": "Erik"
}
POST /api/mobile/position
Alternativ endpoint för positionsuppdatering (samma format).

NOTIFIKATIONER
GET /api/mobile/notifications?unread=true&limit=50
Hämta notifikationer. Svar:

{
  "notifications": [{ "id": "...", "type": "job_assigned", "title": "Ny order", "message": "...", "isRead": false, "createdAt": "..." }],
  "unreadCount": 3,
  "total": 15
}
PATCH /api/mobile/notifications/:id/read
Markera en notis som läst.

PATCH /api/mobile/notifications/read-all
Markera alla som lästa.

GET /api/mobile/notifications/count
Snabb räknare: { "unreadCount": 3 }.

OFFLINE SYNC (Outbox-mönster)
Appen ska ha IndexedDB/AsyncStorage-baserad offlinekö. Vid återanslutning skickas allt i ett batch-anrop.

POST /api/mobile/sync
{
  "actions": [
    { "clientId": "uuid-1", "actionType": "status_update", "payload": { "orderId": "...", "status": "completed" } },
    { "clientId": "uuid-2", "actionType": "note", "payload": { "orderId": "...", "text": "..." } },
    { "clientId": "uuid-3", "actionType": "deviation", "payload": { "orderId": "...", "description": "...", "severity": "medium", "category": "other" } },
    { "clientId": "uuid-4", "actionType": "material", "payload": { "orderId": "...", "articleId": "...", "quantity": 2 } },
    { "clientId": "uuid-5", "actionType": "gps", "payload": { "latitude": 57.7, "longitude": 11.9 } },
    { "clientId": "uuid-6", "actionType": "inspection", "payload": { "orderId": "...", "inspections": [{ "category": "...", "status": "ok" }] } }
  ]
}
Svar: { success, processed, completed, failed, results: [{ clientId, status, error? }] }

GET /api/mobile/sync/status
Hämta synkstatus och senaste logg.

AI-FUNKTIONER
POST /api/mobile/ai/chat
AI-chatt (GPT-4o-mini, svenska):

{ "message": "Hur hanterar jag felaktigt avfall?", "context": { "orderNumber": "WO-001", "customerName": "BRF Norden" } }
Svar: { "response": "..." }

POST /api/mobile/ai/transcribe
Röst till text (Whisper, svenska):

{ "audio": "<base64-webm>" }
Svar: { "text": "..." }

POST /api/mobile/ai/analyze-image
Bildanalys (GPT-4o-mini vision):

{ "image": "<base64-jpeg>", "context": "Kontrollera kärlets skick" }
Svar: { "category": "damaged_container", "description": "Spricka synlig i plastkärlet", "severity": "medium" }

FOTOUPLADDNING (Presigned URL)
POST /api/field-worker/tasks/:id/upload-photo
Genererar en presigned upload-URL. Svar: { uploadURL, objectPath, workOrderId }

Steg:

Anropa ovan endpoint → få uploadURL
PUT bilden direkt till uploadURL
Bekräfta med:
POST /api/field-worker/tasks/:id/confirm-photo
{ "objectPath": "...", "category": "before" }
VÄDER
GET /api/mobile/weather?lat=57.7&lon=11.9
Hämtar väderprognos från Open-Meteo. Ingen auth krävs.

DAGLIG SAMMANFATTNING
GET /api/mobile/summary
Svar: { totalOrders, completedOrders, remainingOrders, totalDuration }

DATAMODELL – Work Order (viktiga fält)
id, tenantId, objectId, resourceId, title, description,
scheduledDate, scheduledStartTime, scheduledEndTime,
estimatedDuration, actualStartTime, actualEndTime,
status (planned|dispatched|on_site|in_progress|completed|failed|cancelled),
priority (low|normal|high|urgent),
address, latitude, longitude, what3words,
notes (JSON array), deviations (JSON array), materialsUsed (JSON array),
metadata (JSON - innehåller subSteps, completedSubSteps, photos),
articleId, quantity, unit
REALTIDSNOTISER (WebSocket)
Backend har WebSocket-notiser. Fältappen kan lyssna på förändrade ordrar, nya ordrar, avbokningar och schemaändringar via WebSocket. Token-baserad auth krävs (begärs via admin-gränssnitt).
