Task #64: Planerarvy & Ändringshantering för kundrapporter
Backend (server/routes/portalRoutes.ts + server/storage.ts)
GET /api/customer-change-requests

Returnerar { items, total } med server-side paginering (limit/offset, default 50, max 100)
Filtrering på: status, category, customerId, objectId, dateFrom, dateTo
Enrichar varje rapport med objectName, objectAddress och customerName via objekts- och kunduppslag
Robust param-parsing (skyddar mot NaN/negativa värden)
GET /api/customer-change-requests/summary

Returnerar aggregerade statusräknare (new, reviewed, resolved, rejected, total) oberoende av paginering
Stödjer samma filter som huvudendpointen (utom status, eftersom den aggregerar alla statusar)
Används av KPI-korten i frontend
GET /api/customer-change-requests/counts-by-object

Returnerar Record<objectId, antal> för rapporter med status new
Används för att visa badge-räknare på ObjectsPage
PATCH /api/customer-change-requests/:id/status

Uppdaterar status + granskningskommentar (reviewNotes)
Sätter reviewedBy och reviewedAt
POST /api/customer-change-requests/:id/create-work-order

Skapar en arbetsorder (typ service, status draft) kopplad till rapportens objekt och kund
Titel: "Kundrapport: [kategorinamn]", beskrivning inkluderar rapporttext + rapport-ID
Sätter rapporten till resolved med reviewNotes "Arbetsorder skapad: [workOrderId]"
Idempotency-skydd: avvisar med 409 om rapporten redan konverterats
Skickar WebSocket-notifikation till tenant
