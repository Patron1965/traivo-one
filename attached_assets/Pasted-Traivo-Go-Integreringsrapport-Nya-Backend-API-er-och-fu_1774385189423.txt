1. Autentisering — Mobil Token Bridge
Traivo Go kan autentisera via PIN/lösenord som förut, men det finns nu också en webb-brygga:

Befintlig login:

POST /api/mobile/login
Body: { email, pin } eller { username, password }
Response: { token, resourceId, expiresIn }
Token-hantering:

Alla mobil-API-anrop kräver header: Authorization: Bearer <token>
Token gäller 24 timmar
req.mobileResourceId identifierar den autentiserade resursen i alla endpoints
2. NYA Endpoints — Avvikelser & Kundrapporter
Skapa avvikelse på en order:

POST /api/mobile/orders/:id/deviations
Body: { type, description, severity, category, photos[] }
Auth: Bearer token (order.resourceId måste matcha req.mobileResourceId)
Hämta mina avvikelser:

GET /api/mobile/deviations/mine
Auth: Bearer token (filtrerar på resource.name matchning mot reportedByName)
Kundrapporter (Change Requests):

POST /api/mobile/customer-change-requests
Body: { category, description, severity, objectId, customerId, photos[] }
GET /api/mobile/customer-change-requests/mine
GET /api/mobile/customer-change-requests/categories
Foto-upload via presigned URL (tvåstegs-flow):

POST /api/mobile/customer-change-requests/upload-photo
Response: { uploadUrl, filePath }
POST /api/mobile/customer-change-requests/confirm-photo  
Body: { filePath }
3. NYA Endpoints — Fältarbetaruppgifter (Task Dependencies)
GET /api/field-worker/tasks?date=YYYY-MM-DD&resourceId=xxx
POST /api/field-worker/tasks/:id/start
POST /api/field-worker/tasks/:id/complete
POST /api/field-worker/tasks/:id/update-metadata
Dessa stödjer beroendehantering mellan uppgifter (en uppgift kan blockeras tills en annan är klar).

5. Ny funktionalitet i SimpleFieldApp (referens för Go)
Dessa funktioner finns i webb-appen och bör implementeras/uppdateras i Traivo Go:

Uppgiftssammanfattning: Aggregerade räkningar ovanför jobblistan ("3 tvättar, 2 besiktningar")
Nästa stopp-kort: Navigation med avstånd/tid per jobb, deep link till Apple/Google Maps
Planerade anteckningar: plannedNotes-fält på work orders — planerarens meddelande till fältarbetaren
10-minuters varning: Toast/notis 10 min innan nästa schemalagt jobb
Carry-over logik: POST /api/work-orders/carry-over — flytta gårdagens oavslutade jobb till idag
Route GPS start: Ruttoptimering använder live GPS-position (om <30 min gammal) istället för hemadress
Smart kontrollmallar: AI-föreslagna checklista-steg baserade på ordertyp och historik
WebSocket-notiser: Anslut till /ws/notifications med Bearer token för realtidsuppdateringar (job_assigned, job_cancelled, schedule_changed)
